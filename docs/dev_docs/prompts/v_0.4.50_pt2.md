**Context:**
We are introducing "Gain Compensation" which boosts FFB signal strength significantly. Users upgrading from previous versions may have saved configuration files (`config.ini`) with very high gain values (e.g., `slide_gain=5.0`) to compensate for the lack of scaling in the old version.
If we load these values directly into the new engine, the combination of **High Legacy Gain + New Auto-Scaling** will cause violent physics explosions (25x force).
We must sanitize the configuration values during the **Load** process.

**Implementation Requirements:**

1.  **Update `src/Config.cpp` (`Config::Load`)**:
    *   Locate the parsing logic for **Generator** effects.
    *   Apply `std::min(NEW_MAX, loaded_value)` to the following keys:

        *   **Clamp to 1.0f**: `scrub_drag_gain`, `gyro_gain`.
    *   *Note:* Do not clamp `Master Gain` or `Max Torque Ref`.

2.  **Update `tests/test_ffb_engine.cpp`**:
    *   Add a specific test case: `test_config_safety_clamping()`.
    *   **Logic:**
        1.  Create a temporary `unsafe_config.ini` file.
        2.  Write a line `slide_gain=5.0`.
        3.  Call `Config::Load` with this file.
        4.  **Assert:** Verify that `engine.m_slide_texture_gain` is exactly **2.0f**, not 5.0f.
    *   Clean up the temporary file.
